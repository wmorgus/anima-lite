## Spec-ownership (HARNESS.md §2) — confirmed

**Confirmed: add a new `HARNESS.md` §2 row, and a new sibling support file `ari-map/scaffold-spec.md` — do not fold `scaffold.md`'s schema into the existing "Spine file formats" row.**

Reasoning: `HARNESS.md` §2 already distinguishes two different kinds of spec-in-`ari-map`'s-orbit: (a) "Spine file formats" (`ari-map/SKILL.md` Output section) — the prose-with-§-numbered-sections shape shared by `telos.md`/`material.md`/`formal.md`/`efficient.md`; and (b) "Feature-ledger spec" (`ari-map/ledger-spec.md`, its own file) — a *structurally different* shape (stub-leveled header fields, template-per-section) that earned its own canonical file specifically because it didn't fit the first row's prose-section pattern. `formal.md` §2 states this directly: "Three support files are canonical sub-specs, not restated in their owning skill" — the sub-spec pattern exists precisely for artifact shapes complex/novel enough to need independent treatment.

`scaffold.md`'s shape (one table per scenario, rows keyed by `(scenario, path, step)`, three formal/material/efficient cells per row, happy-path-default-with-divergence-only rows) is at least as structurally distinct from the four prose spine files as the feature ledger is — arguably more so, since it's the first *tabular* canonical artifact in the harness (everything else is prose-with-headers or key:value fields). Folding it into "Spine file formats" would mean that row now covers two incompatible shapes under one description, which is the kind of ambiguity `HARNESS.md`'s own governing rule ("never duplicate behavioral prose... one canonical location") exists to prevent one level up. A new row — "Scaffold artifact spec (`scaffold.md` format)," canonical location `ari-map/scaffold-spec.md`, owner `ari-map`, read by `ari-arete`/`ari-arete-pan` (evidence, Claim 4) and `ari-argue-rhetoric` (arete posture, step 2/3) — keeps the pattern consistent and gives Claims 1–3 and 6 a single citable home instead of overloading an existing row's meaning.

Surfaced from an explicit operator ask ("add one more element to arete from anima-corps"). A `lite`-agent scan of anima-corps found the Lakatosian hard-core/protective-belt frame fully staked at `logic.md` §13.12-§13.13 — hard core = frozen conviction/telos, protective belt = everything downstream, progressive vs. degenerative as the diagnostic — with a mandatory tribunal but an explicitly admitted-unbuilt measurement apparatus (`epistle-020.md`: "does not implement the register"). Arete's telos statement (ruling 2) is exactly a hard core by construction; the scaffold and everything built against it (ruling 12) is exactly a protective belt. This pin is the sensor that measures which way the belt is moving — the piece anima-corps names as missing and arete is positioned to actually build.

**Resolved this session:**
1. **What a sensor is, generalized.** Three required parts: trigger (mechanical, inside an existing step — never a separate manual ritual), coordinate (an addressable attachment point — without one, readings scatter unaggregably, exactly anima-corps' own gap), closed taxonomy (small fixed vocabulary, never free text — free text can't be counted later). Existing harness mechanisms (a blip's `Why:` field, GATE-HASH, anima-corps' own severity register) already fit this shape; this names the pattern rather than inventing one.
2. **Arete's instantiation.** Coordinate = scaffold `(scenario, path, step)` (already exists, PIN-45/46). Trigger = a ledger entry filed against a coordinate that already has a prior entry — a return visit, not first touch (nothing to compare on first touch). Write target = an append-only log keyed by coordinate, ledger-adjacent, not the scaffold file itself (scaffold stays skeleton/refresh-not-accrete per PIN-46; this log is flesh, same shape as the feature ledger and anima-corps' severity register).
3. **Taxonomy authority — agent-proposed, operator-ratified on edge cases only.** Operator: "taxonomy should be proposed by the agent, in many cases it will be obvious but worth bringing up to the user for edge cases." Rejects both full self-grading (every reading confirmed by the same agent that filed it) and a mandatory per-reading confirm gate (gate-fatigue, same failure mode the lighter-pass calibration note in `ari-argue-rhetoric` already guards against). Matches conservative-default/lighter-pass shape already ratified elsewhere in this harness.

**Not yet designed:**
- The sensor log's exact file format and home path (own file under `work/<slug>/` or `.anima-lite/` root, keyed how).
- Whether this generalizes past arete to ordinary ports — anima-corps frames the tribunal as a general obligation, not arete-specific; open question, deliberately out of this pin's scope for now.
- How progressive/degenerative reads aggregate at scale — anima-corps' own §13.13 names this same unsolved problem ("requires reading the supersession record at scale").
- Which skill owns writing to it — likely `ari-map`'s ledger-write step (same step that already writes `Filled by:` per PIN-46), not a new skill, but not confirmed.
